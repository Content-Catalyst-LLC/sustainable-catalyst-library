# Accession and Custody Guide — v3.6.0

## Accession purpose

An accession records the administrative event through which materials enter institutional custody.

It is not the same as an archival series or item description.

## Required fields

Recommended minimum:

```text
Accession number
Accession date
Receiving collection
Transfer or acquisition method
Source or transferring body
Extent
Processing status
```

## Sensitive fields

Donor names, agreements, restrictions, and custody notes are private by default.

## Custody events

Each event can store:

```text
Date
From
To
Method
Note
Stable event UUID
```

Events are ordered by date.

## Born-digital intake

Recommended steps:

1. isolate the received media or files;
2. document the transfer source;
3. record the original package;
4. scan for malware using appropriate institutional tools;
5. create checksums;
6. retain originals;
7. create preservation and access copies;
8. record format and size;
9. link the digital objects to an archive component;
10. update processing status.

## Quarantine

Use Quarantined when materials have not completed security, privacy, rights, or preservation review.

Quarantine status does not automatically restrict a collection; access controls must still be recorded.

## Agreement review

Before public release, review:

- ownership;
- copyright;
- privacy;
- donor restrictions;
- access conditions;
- permitted reuse;
- required attribution;
- embargo terms;
- disposition obligations.
