# Evidence and Claim Workflow — Knowledge Library v2.7.0

## Recommended workflow

### 1. Create or locate the Source

Use the Citation Manager or Source Discovery workspace to create a reviewed Research Source.

### 2. Create an Evidence Note

Open:

```text
SC Library → Evidence Notes → Add Evidence Note
```

Use:

- the title for a short stable label
- the main editor for the exact quotation, paraphrase, data point, definition, method, or observation
- the excerpt for a concise research note

### 3. Record the locator

Select the locator type and record:

```text
start
end
custom label
```

Use a custom label for nonstandard locators such as:

```text
Appendix B
Equation 4
Dataset row 218
Interview transcript line 84
```

### 4. Preserve context

Record context before and after the evidence when removing the evidence from its surrounding passage could change its meaning.

### 5. Add analysis

The analytical note should explain relevance without rewriting the evidence itself.

Keep distinct:

```text
Evidence text
Research note
Interpretation
Claim-link rationale
```

### 6. Verify

Check:

```text
Exact wording or transcription checked
Locator checked against source material
```

Changing citation-critical fields clears prior verification unless re-verification is confirmed.

### 7. Create a Research Claim

Open:

```text
SC Library → Research Claims → Add Research Claim
```

Use the main editor for the complete claim statement.

Record scope, assumptions, uncertainty, limitations, and plausible counterclaims.

### 8. Link evidence

Return to the Evidence Note and add the Research Claim.

Choose the relationship and strength.

A relationship describes how the evidence bears on that particular claim. It is not a general rating of the Source.

### 9. Review counterevidence

A claim packet should include supporting, qualifying, contextual, contradictory, and unresolved evidence.

A high support count does not replace review of source quality, methodology, independence, and relevance.

### 10. Publish selectively

Private is the default.

Use Public only when:

- quotation rights and publication context have been reviewed
- the Source or document is public
- the evidence is appropriate to publish
- claim limitations are visible
- the project permits public evidence

## Workspace

Open:

```text
SC Library → Evidence and Claims
```

The workspace summarizes notes, claims, links, and verified evidence.

## Cross-record panels

Research Source editor:

```text
Quotations and Evidence
```

Research Project editor:

```text
Claims and Evidence Packet
```

## Citation-ready export

An Evidence Note generates:

- quotation plus Harvard in-text citation
- Markdown evidence note
- Source reference
- locator
- claim relationships

A Claim generates a relationship-grouped evidence packet.

A Research Project generates a multi-claim evidence packet.


## Private shortcode previews

Shortcodes are public-only by default.

Authorized editors can explicitly preview private records with:

```text
include_private="true"
```

Do not place private-preview shortcodes on cacheable public pages.
