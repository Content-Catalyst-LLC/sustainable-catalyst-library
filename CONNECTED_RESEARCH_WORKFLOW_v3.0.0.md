# Connected Research Workflow — Knowledge Library v3.0.0

## 1. Create the Research Project

Open:

```text
SC Library → Research Projects → Add Research Project
```

Record the project title, summary, code, visibility, status, citation style, and bibliography title.

## 2. Define the project brief

Use **Connected Project Brief** to record:

- research question
- objectives
- methods
- scope
- start and target dates

Keep the research question stable enough to guide source inclusion decisions.

## 3. Discover Sources

From the project editor or Research Environment choose:

```text
Discover Sources for Project
```

Search Crossref, OpenAlex, DataCite, PubMed, PubMed Central, Library of Congress digital collections, Open Library, and Google Books.

Google Scholar and WorldCat remain browser-search handoffs.

Imported Sources enter the project as Candidates rather than being included automatically.

## 4. Review Source metadata

Before inclusion:

- confirm title, authors, organization, date, and publication details
- validate DOI, ISBN, PMID, URL, and access location
- review duplicate warnings
- review citation completeness and reliability
- verify metadata where appropriate

## 5. Organize the bibliography

For each Source choose:

- project role
- bibliography section
- inclusion status
- priority
- project annotation

A Source may serve a different role in different projects.

## 6. Connect documents

Attach Knowledge Library documents directly to the project.

Documents related to project Sources are also surfaced automatically.

## 7. Develop claims and evidence

Use v2.7.0 Research Claims and Evidence Notes.

Project packets connect:

```text
Sources → quotations/evidence → claims → project
```

Record supporting, qualifying, contextual, contradictory, and unresolved evidence.

## 8. Monitor health

Use the readiness panel to identify:

- incomplete citations
- unverified metadata
- duplicate warnings
- missing access locations
- absent claims, evidence, or documents

The score is an operational checklist, not a research-quality verdict.

## 9. Create bibliography snapshots

Create a named snapshot before:

- a major literature-review revision
- a report draft
- peer review
- publication
- a project handoff

## 10. Export

Use:

```text
Markdown for research drafts
HTML for web or document workflows
BibTeX for LaTeX and reference managers
RIS for broad citation-manager exchange
CSL JSON for structured citation workflows
Connected JSON for platform handoffs
```

## 11. Publish selectively

Public shortcodes include only records permitted by the project, Source, document, claim, and evidence visibility boundaries.

Team records, activity history, snapshots, and private Source metadata remain private.
