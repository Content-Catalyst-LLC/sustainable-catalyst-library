# Platform Research Bundle Format — v3.4.0

## Schema

```text
sc-platform-research-bundle/1.0
```

## Top-level fields

```text
schema
bundle_id
handoff_uuid
handoff_id
source_product
target_product
handoff_type
target_contract
project_identity
sections
request
adapter
context
return
provenance
```

## Context sections

### project

Connected Research Project brief, question, objectives, methods, scope, dates, team, documents, and Sources where available.

### bibliography

Project bibliography, sections, citations, annotations, and Source metadata generated by the v3.0.0 environment.

### evidence_packet

Claims, Evidence Notes, quotations, Source relationships, and project evidence generated by the v2.7.0 system.

### semantic_context

Project Topics, Concepts, Named Entities, typed relationships, and v3.2.0 project coverage.

### pathways

Pathways containing the Project and recommendation results from v3.3.0.

### source_integrity

Project-specific correction, supersession, expression-of-concern, retraction, and replacement review from v3.1.0.

### dataset_references

Portable provider and dataset identifiers, URLs, geography, time range, and notes.

## Integrity checksum

The stored bundle checksum is:

```text
SHA-256(JSON(bundle))
```

Consumers should compare the received bundle to the advertised checksum before import.

## ZIP export

The ZIP is a metadata and context bundle. It does not automatically copy private PDFs or binary attachments.

This avoids uncontrolled duplication and preserves access boundaries. Attachment identifiers and URLs may be present in structured Source or document metadata.
