# Research Project to Knowledge Pathway Workflow — v3.3.0

## Generate a draft

Open:

```text
SC Library → Pathways and Maps
```

Select a Research Project and choose **Create Draft Pathway**.

The generated pathway remains Draft.

## Generated sequence

The draft can include:

1. the Research Project record as Orientation;
2. connected Knowledge Library documents as Foundation;
3. included and candidate Research Sources as Sources and Evidence;
4. Research Claims as Analysis and Critique;
5. Evidence Notes as optional Analysis and Critique steps.

## Retained context

The pathway inherits:

- project title and summary;
- project description;
- project objectives as pathway outcomes;
- project Knowledge Topics;
- project-to-pathway provenance.

## Editorial review

Before publication:

- remove administrative or private records;
- decide which candidate Sources belong in the route;
- reorder records into a readable sequence;
- add transition notes;
- classify the audience and level;
- divide long routes into prerequisites and continuations;
- review Source integrity warnings;
- confirm that public Evidence Notes are appropriate for readers.

## Regeneration

By default, a project returns its existing derived pathway.

Use a forced new pathway only when a genuinely separate route is needed. The system does not automatically synchronize an existing pathway every time the Research Project changes.
