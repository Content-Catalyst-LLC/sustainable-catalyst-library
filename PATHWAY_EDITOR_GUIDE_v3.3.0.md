# Knowledge Pathway Editor Guide — v3.3.0

## Create a pathway

Open:

```text
SC Library → Knowledge Pathways → Add Knowledge Pathway
```

Use the title for the public route name, the excerpt for a concise summary, and the editor for the pathway introduction.

## Define the audience and level

Choose one level:

```text
Introductory
Foundational
Intermediate
Advanced
Expert or specialist
Mixed levels
```

Describe the intended audience directly. Examples:

```text
New readers of climate governance
Graduate researchers
Municipal sustainability practitioners
Technical reviewers
Institutional decision-makers
```

## Write outcomes

Enter one outcome per line. Outcomes should describe what a reader can understand, evaluate, compare, or produce after completing the pathway.

## Add steps

Each step requires:

- record type;
- record ID;
- stage;
- difficulty;
- required or optional state.

Optional fields:

- display-label override;
- time estimate;
- purpose or transition note.

Use **Find** to search the selected record type.

## Order steps

Drag the step rows using the reorder handle. The saved row order is the canonical pathway order.

A record can appear only once in a pathway.

## Use stages

Stages organize the route visually and intellectually. They do not override the saved sequence.

A common learning route:

```text
Orientation → Foundation → Core Reading → Application → Analysis → Synthesis → Further Exploration
```

A research route may emphasize:

```text
Orientation → Sources and Evidence → Analysis and Critique → Synthesis
```

## Connect pathways

Use prerequisite pathways for material a reader should complete first.

Use continuation pathways for logical next routes.

Do not create circular prerequisite structures intentionally. v3.3.0 prevents direct self-links but does not attempt to prove that a multi-pathway graph is acyclic.

## Add Topics, Concepts, and Entities

Knowledge Topics support subject discovery.

Concepts support framework, theory, method, model, and principle matching.

Named Entities support organizations, places, treaties, laws, standards, datasets, technologies, instruments, and other identifiable records.

These assignments inform Research Librarian recommendations.

## Publish

Preview the pathway and article map before publication.

Confirm:

- every required record is public;
- private Evidence Notes are not expected to appear publicly;
- prerequisite and continuation pathways are published;
- time estimates are credible;
- the sequence is understandable without the visual map;
- recommendation terms are specific and not promotional.
