# Document Family Editorial Guide — v2.3.0

A document family is a public repository section, not a blog category.

## Recommended family record

```text
Name
Public kicker
Editorial description
Repository order
Featured-family setting
Published document assignments
```

## Editorial pattern

Use a short kicker to establish the institutional purpose:

```text
Institutional Foundations
Research Publications
Methods and Evidence
Governance Library
Technical Library
Institutional Archive
```

Use the Description field for two or three concise sentences explaining:

- What the family contains
- Why the records belong together
- Whether historical or superseded material is included

## Document assignment

A document should normally have:

- One primary document family
- One primary document type
- A lifecycle state
- A version when versioning is meaningful
- A publication date when known
- A concise listing summary
- The original PDF attachment
- Reviewed readable document content

Families organize the institutional repository. Types describe what kind of document each record is.
