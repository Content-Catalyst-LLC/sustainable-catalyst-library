# Institutional Permissions and Governance — v4.0.0

## Capability groups

### Read

```text
sc_library_read_institutional
```

### Operate records

```text
sc_library_manage_institutional_records
sc_library_publish_institutional
sc_library_export_institutional
```

### Administer institutions

```text
sc_library_manage_institutional
sc_library_manage_institutional_permissions
```

### Coordinate platform workflows

```text
sc_library_manage_institutional_handoffs
sc_library_view_institutional_health
```

## Default role mapping

Administrator: all institutional capabilities.

Editor: read, record management, publishing, handoffs, health, and export.

Author: institutional read.

Review role mappings after installation. Existing custom roles are not automatically granted capabilities.

## Governance guidance

- Assign a responsible steward.
- Use Restricted for confidential or sensitive records.
- Use Institutional for internal operational records.
- Use Research unit where access policy is managed by a unit-specific workflow.
- Mark Approved only after the relevant review or governance process.
- Mark Published only when the record is intentionally released.
