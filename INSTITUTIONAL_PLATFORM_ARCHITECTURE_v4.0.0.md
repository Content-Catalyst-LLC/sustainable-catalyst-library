# Institutional Platform Architecture — v4.0.0

## Layers

```text
Public portal and institutional workspace
Unified search, graph, health, and handoffs
Institutional identity, units, permissions, and registry
Documents, sources, claims, projects, semantics, pathways, collections
Quality, intelligence, review, publication, export, and federation
WordPress persistence and public API
```

## Principle

v4.0.0 is an orchestration and governance release. It does not replace the specialized v2–v3 systems. It gives them a common identity, institutional context, permission model, search contract, graph contract, health view, and handoff envelope.

## Institutional identity

An institution represents the responsible organization. A research unit represents a laboratory, program, archive, publication unit, platform team, or advisory practice within or associated with an institution.

## Record identity

Each registered record receives a UUID and URN:

```text
urn:sc-library:<site-hash>:<record-type>:<uuid>
```

The site hash separates identities generated by different installations.

## Data boundaries

Visibility:

```text
Public
Institutional
Research unit
Restricted
```

Governance state:

```text
Unclassified
Draft
Managed
Under review
Approved
Published
Archived
Restricted
```

Visibility controls access. Governance state communicates lifecycle and stewardship. Neither field alone establishes legal, ethical, or scholarly approval.
