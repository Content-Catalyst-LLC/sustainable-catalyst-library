# Review Note and Resolution Guide — v3.8.0

## Note types

```text
Comment
Question
Required change
Suggestion
Citation issue
Evidence issue
Integrity issue
Accessibility issue
```

## Severity

```text
Info
Low
Medium
High
Critical
```

High and Critical unresolved notes block review approval.

## Anchoring

A note can reference:

- document ID;
- section label;
- local anchor;
- short quotation;
- parent note;
- assignee.

Do not copy long copyrighted passages into review notes.

## Resolution states

```text
Open
In progress
Resolved
Accepted risk
Dismissed
```

## Resolution record

A useful resolution states:

- what changed;
- where it changed;
- who verified the change;
- whether the document snapshot must be refreshed;
- why risk was accepted or a note was dismissed.

## Snapshot consequence

Any document revision after snapshot causes the review to report Changed until snapshots are deliberately refreshed.

This protects against approving a document version that reviewers did not evaluate.
